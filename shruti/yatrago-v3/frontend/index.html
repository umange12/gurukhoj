⚙ DSA Implementations

All 8 algorithms running live inside this project

🏔
Min-Heap (Priority Queue)
O(log n) insert & extract-min
When you book a seat, your passenger profile is scored: Disability=1 (highest), Women 40+=2, Senior 60+=3, General=4. The Min-Heap always extracts the highest priority passenger first, ensuring fair seat assignment.
📍 Used in: Seat assignment on every booking + Booking waitlist queue
🗺
Dijkstra's Algorithm
O((V+E) log V) shortest path
City connections are stored as a weighted directed Graph (adjacency list). Dijkstra's uses a Min-Heap internally to always pick the cheapest unvisited city next, finding the cheapest/fastest multi-hop route.
📍 Used in: Route Finder below — try Delhi → Goa!
🔀
Merge Sort
O(n log n) stable sort
Every search result is sorted using our custom Merge Sort implementation in server.js. It divides the results array in half recursively, sorts each half, then merges them in order. Stable — equal prices maintain original order.
📍 Used in: All flight/train/bus search results before response
🔎
Binary Search
O(log n) lower & upper bound
After Merge Sort, the results array is price-sorted. Binary Search finds the lower bound (first item ≥ minPrice) and upper bound (first item > maxPrice) in O(log n) — much faster than O(n) linear scan for price range filter.
📍 Used in: Price filter slider on results page
📚
Stack (LIFO)
O(1) push/pop, max 10 items
Every search you do is pushed onto your personal Stack. Newest search = top of stack. Max 10 items — when 11th search comes, oldest is removed (like a circular stack). Duplicates auto-removed before push.
📍 Used in: Search history (login + search to see yours)
🗂
Hash Map
O(1) lookup, prefix indexing
200+ city names are stored in a custom HashMap with prefix indexing. When you type "Del", the prefix key "del" maps to ["Delhi"]. Lookup is O(1). No database query needed — all in-memory at server start.
📍 Used in: City autocomplete on search inputs
🌐
Graph (Adjacency List)
O(V+E) space, directed weighted
Cities are graph nodes. Each flight/train/bus route is a directed weighted edge. Three separate graphs exist: flight-graph, train-graph, bus-graph. Dijkstra's runs on each to find transport-specific shortest paths.
📍 Used in: Route Finder — city connections from DB
💰
Dynamic Pricing (Greedy)
O(1) — real airline logic
Prices increase as departure approaches (like real airlines). <2hrs = 2.2x, <6hrs = 1.8x, <12hrs = 1.4x, <24hrs = 1.2x. Seats <20% remaining also adds surcharge. Early booking = cheapest price!
📍 Used in: All flight/train/bus prices on search results
